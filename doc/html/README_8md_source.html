1 # Description
2 
3 Merlin is a general purpose `C++` library that implements state-of-the-art exact
4 and approximate algorithms for probabilistic inference over graphical models
5 including both directed and undirected models (e.g., Bayesian networks,
6 Markov Random Fields). It can be used for many applications and research in
7 bioinformatics, computer vision, or speech and language processing to name a
8 few. Merlin supports the most common probabilistic inference tasks such as
9 computing the partition function or probability of evidence (PR), posterior
10 marginals (MAP), as well as MAP (also known as maximum aposteriori or most
11 probable explanation) and marginal MAP configurations.
12 
13 Merlin implements the classic Loopy Belief Propagation (LBP) algorithm as well
14 as more advanced generalized belief propagation algorithms such as Iterative
15 Join-Graph Propagation (IJGP) and Weighted Mini-Bucket Elimination (WMB). The
16 default algorithm for computing all four probabilistic inference tasks (PR, MAR,
17 MAP, MMAP) is WMB(i). The algorithm is parameterized by an i-bound that allows
18 for a controllable tradeoff between accuracy of the results and the computational
19 cost. Larger values of the i-bound typically yield more accurate results but
20 it takes more time and memory to compute them. Selecting a large enough i-bound
21 allows for exact inference (i.e., i-bound equal to the treewidth of the model).
22 For relatively small i-bounds Merlin performs approximate inference.

25 # API
26 
27 ## Basic (Black-Box) API
28 Merlin exposes (see `merlin.h` header) a single C-like function called `run()`:
29 
30  int run(const char* inputFile,
31  const char* evidenceFile,
32  const char* queryFile,
33  const char* outputFile,
34  const char* task,
35  const unsigned int ibound,
36  const unsigned int iterations);
37 
38 where:
39 * `inputFile` - name of the file containing the input graphical model specified in the UAI format (see main documentation for details on the format).
40 * `evidenceFile` - name of the file containing the evidence or observations as variable and value pairs (variable indeces are the same as in the `inputFile`).
41 * `queryFile` - name of the file containing the query variable for the marginal MAP inference task (variable indeces are the same as in the `inputFile`).
42 * `outputFile` - name of the file containing the result of the inference task (see main documentation for details on the format).
43 * `task` - name of the probabilistic inference task. The following names can be used: PR, MAR, MAP, MMAP.
44 * `ibound` - parameter i-bound of the inference algorithm `WMB(i)` controling the accuracy of the results. The value should be greater or equal to 2 (default is 2).
45 * `iterations` - parameter iterations of the inference algorithm `WMB(i)` specifing the number of iterations to be executed (default is 100).
46 
47 The function returns 0 if successful, in which case the output file contains the
48 solution to the probabilistic inference task. Otherwise, the return value is 1,
49 in which case the content of the output file is undefined and it should be
50 ignored.

53 ### Example
54 
55 A simple application that uses the library is included with the distribution
56 (see the `demo.cpp` file in the `example/` folder). The source code is given
57 below. The input graphical model is given in the `pedigree1.uai` file, while
58 `pedigree1.evid` contains a set of observations (ie, variable value pairs).
59 In addition, the file `pedigree1.map` contains the query variables for the
60 marginal MAP task. The application solves the following inference tasks (the
61 results are written in the corresponding `.out` files):
62 
63 * MAR = posterior marginals (ie, marginal distributions of all unobserved variables given the evidence)
64 * MAP = maximum aposteriori (ie, configuration of unobserved variables having maximum probability)
65 * MMAP = marginal MAP (ie, configuration of the query variables having maximum marginal probability)
66 
67  #include "merlin.h"
68 
69  void demo_run(void) {
70 
71  // Init parameters
72  unsigned int ibound = 4;
73  unsigned int iterations = 300;
74  const char* inputFile = "pedigree1.uai";
75  const char* evidenceFile = "pedigree1.evid";
76  const char* queryFile = "pedigree1.map";
77  const char* outputFileMAR = "pedigree1.MAR.out";
78  const char* outputFileMAP = "pedigree1.MAP.out";
79  const char* outputFileMMAP = "pedigree1.MMAP.out";
80 
81  // MAR task
82  run(inputFile, evidenceFile, "", outputFileMAR, "MAR", ibound, iterations);
83 
84  // MAP task
85  run(inputFile, evidenceFile, "", outputFileMAP, "MAP", ibound, iterations);
86 
87  // MMAP task
88  run(inputFile, evidenceFile, queryFile, outputFileMMAP, "MMAP", ibound, iterations);
89 
90  }
91 
92 The application should be compiled with the `-lmerlin` option. It is also
93 possible to just run the `example` wrapper script (generated automatically
94 during by the build scripts) in the `example/` folder.

97 ## Advanced API
98 Class `Merlin` defined in `merlin.h` header exposes most of the functionality
99 of the library. A graphical model is a collection of factors (or positive
100 real-valued functions) defined over subsets of variables. Variables are
101 assumed to be indexed from `0`.
102 
103 ### Methods
104 
105  bool read_model(const char* f)
106 This method loads the graphical model from a file which is specified using the
107 UAI format (see also the File Formats section). Returns `true` if successful
108 and `false` otherwise.
109 
110  bool read_evidence(const char* f)
111 This method loads the evidence variables and their corresponding observed values
112 from a file which is also specified using the UAI format. Returns `true` if
113 successful, and `false` otherwise.
114 
115  bool read_query(const char* f)
116 This method loads the query variables from a file specified using the UAI format.
117 The query variables (also known as MAX of MAP variables) are only specific to
118 Marginal MAP (MMAP) inference tasks. Returns `true` if successful, and `false`
119 otherwise.
120 
121  bool set_task(size_t t)
122 This method sets the probabilistic inference task to be solved. The possible
123 values for the `t` parameter are:
124 * `MERLIN_TASK_PR` : Partition function (probability of evidence)
125 * `MERLIN_TASK_MAR` : Posterior marginals (given evidence)
126 * `MERLIN_TASK_MAP` : Maximum aposteriori (given evidence)
127 * `MERLIN_TASK_MMAP` : Marginal MAP (given evidence)
128 
129  bool set_algorithm(size_t a)
130 This method sets the the algorithm to be used when solving the selected
131 probabilistic inference task. The possible values for the `a` parameter are:
132 * `MERLIN_ALGO_GIBBS` : Gibbs sampling
133 * `MERLIN_ALGO_LBP` : Loopy belief propagation
134 * `MERLIN_ALGO_IJGP` : Iterative join graph propagation
135 * `MERLIN_ALGO_JGLP` : Join graph linear programming
136 * `MERLIN_ALGO_WMB` : Weighted mini-bucket elimination
137 * `MERLIN_ALGO_AOBB` : AND/OR branch and bound search (not implemented)
138 * `MERLIN_ALGO_AOBF` : Best-first AND/OR search (not implemented)
139 * `MERLIN_ALGO_RBFAOO` : Recursive best-first AND/OR search (not implemented)
140 
141  void set_param_ibound(size_t ibound)
142 This method sets the i-bound parameter which is used by the following
143 algorithms: `WMB`, `IJGP`, `JGLP` (as well as search based ones `AOBB`, `AOBF`,
144 and `RBFAOO`). The default value is `4`.
145 
146  void set_param_iterations(size_t iter)
147 This method sets the number of iterations to be executed by the inference
148 algorithm. The parameter is specific to the following algorithms: `WMB`, `IJGP`,
149 `JGLP`, and `LBP`. The default value is `100`. For Gibbs sampling consider
150 runnig several thousands of iterations.
151 
152  void set_param_samples(size_t s)
153 This method sets the number of samples to be generated in each iteration of the
154 `GIBBS` sampling algorithm. The default value is `100`.
155 
156  void run()
157 This method runs the inference algorithm for the selected task on the input
158 graphical model and evidence (if any). The output is generated into a file
159 specified using the UAI format. The name of the output file is obtained from
160 the input file augmented with the `task.out` suffix, where `task` corresponds
161 to one of the follwing: `PR`, `MAR`, `MAP`, or `MMAP`.
162 
163 ### Example
164 A simple example using the advanced Merlin API is included in the `example/`folder
165 (i.e., `demo.cpp` file). The source code is given below. The input graphical model
166 is given in the `pedigree1.uai` file, while `pedigree1.evid` contains a set
167 of observations (ie, variable value pairs). In addition, the file `pedigree1.map`
168 contains the query variables for the Marginal MAP task. The application solves all four
169 inference tasks and the results are written in the corresponding `.out` files.
170 
171  #include "merlin.h"
172 
173  void demo_api() {
174 
175  // Init parameters
176  unsigned int ibound = 4;
177  unsigned int iterations = 300;
178  const char* model_file = "pedigree1.uai";
179  const char* evid_file = "pedigree1.evid";
180  const char* query_file = "pedigree1.map";
181 
182  // Initialize the Merlin engine
183  Merlin eng;
184  eng.set_param_ibound(4);
185  eng.set_param_iterations(300);
186  eng.read_model(model_file);
187  eng.read_evidence(evid_file);
188 
189  // Solve a MAR task
190  eng.set_task(MERLIN_TASK_MAR);
191  eng.set_algorithm(MERLIN_ALGO_WMB);
192  eng.run();
193 
194  // Solve a MAP task
195  eng.set_task(MERLIN_TASK_MAP);
196  eng.set_algorithm(MERLIN_ALGO_WMB);
197  eng.run();
198 
199  // Solve a MMAP task
200  eng.read_query(query_file);
201  eng.set_task(MERLIN_TASK_MMAP);
202  eng.run();
203  }

210 # Source Code
211 
212 The source code is organized along the following directory structure and
213 requires a standard GNU build using the GNU Autotools toolchain.
214 
215 * `src/` - contains the source (.cpp) files
216 * `include/` - contains the header (.h) files
217 * `example/` - contains an example program (demo.cpp) that uses the library

219 
220 # Build
221 
222 The simplest way to compile the library is to `cd` to the directory containing
223 the source code and type `./configure`. Then type `make` to compile the library.
224 The binaries will be generated in the default location, `src/.libs/`. `make` will
225 also build the example program in `demo/`. Finally, type `make install` to install the
226 library in the default location (`/usr/lib` for the `libmerlin.so` object and `/usr/include`
227 for `merlin.h`). When installing into a prefix owned by the root, it is recommended
228 that the library be configured and built as a regular user, and only the
229 `make install` phase executed with root privileges (see INSTALL for more details).
230 
231  -$ ./configure
232  -$ make
233  -$ sudo make install
234 
235 The interface of the library is exposed in the `merlin.h` header file which must be
236 included in the source files of the application. See demo/ for a simple example.
237 The name of the shared library is `libmerlin`.

249 # File Formats
250 
251 ## Input File Format
252 
253 Merlin uses a simple text file format which is specified below to describe a
254 problem instances (i.e., graphical model). The format is identical to the one
255 used during the UAI Inference competitions.
256 
257 ### Structure
258 The input file format consists of the following two parts, in that order:
259 
260  <Preamble>
261  <Factors>
262 
263 The contents of each section (denoted <…> above) are described in the following:
264 
265 #### Preamble
266 The description of the format will follow a simple Markov network with three
267 variables and two functions. A sample preamble for such a network is:
268 
269  MARKOV
270  3
271  2 2 3
272  2
273  2 0 1
274  2 1 2
275 
276 The preamble starts with one line denoting the type of network. Generally, this
277 can be either BAYES (if the network is a Bayesian network) or MARKOV (in case of
278 a Markov network).
279 
280 The second line contains the number of variables.
281 
282 The third line specifies the cardinalities of each variable, one at a time,
283 separated by a whitespace (note that this implies an order on the variables
284 which will be used throughout the file).
285 
286 The fourth line contains only one integer, denoting the number of cliques in the
287 problem. Then, one clique per line, the scope of each clique is given as follows:
288 The first integer in each line specifies the number of variables in the clique,
289 followed by the actual indexes of the variables. The order of this list is not
290 restricted (for Bayesian networks we assume that the child variable of the clique
291 is the last one). Note that the ordering of variables within a factor will
292 follow the order provided here.
293 
294 Referring to the example above, the first line denotes the Markov network, the
295 second line tells us the problem consists of three variables, let's refer to
296 them as `X`, `Y`, and `Z`. Their cardinalities are `2`, `2`, and `3`
297 respectively (from the third line). Line four specifies that there are 2 cliques.
298 The first clique is `X,Y`, while the second clique is `Y,Z`. Note that
299 variables are indexed starting with `0`.
300 
301 #### Factors
302 Each factor is specified by giving its full table (i.e, specifying a
303 non-negative real value for each assignment). The order of the factors is
304 identical to the one in which they were introduced in the preamble, the first
305 variable has the role of the 'most significant' digit. For each factor table,
306 first the number of entries is given (this should be equal to the product of the
307 domain sizes of the variables in the scope). Then, one by one, separated by
308 whitespace, the values for each assignment to the variables in the factor's
309 scope are enumerated. Tuples are implicitly assumed in ascending order, with
310 the last variable in the scope as the `least significant`. To illustrate, we
311 continue with our Markov network example from above, let's assume the following
312 conditional probability tables:
313 
314  X | P(X)
315  0 | 0.436
316  1 | 0.564
317 
318  X Y | P(Y,X)
319  0 0 | 0.128
320  0 1 | 0.872
321  1 0 | 0.920
322  1 1 | 0.080
323 
324  Y Z | P(Z,Y)
325  0 0 | 0.210
326  0 1 | 0.333
327  0 2 | 0.457
328  1 0 | 0.811
329  1 1 | 0.000
330  1 2 | 0.189
331 
332 Then we have the corresponding file content:
333 
334  2
335  0.436 0.564
336 
337  4
338  0.128 0.872
339  0.920 0.080
340 
341  6
342  0.210 0.333 0.457
343  0.811 0.000 0.189
344 
345 Note that line breaks and empty lines are effectively just a whitespace,
346 exactly like plain spaces “ ”. They are used here to improve readability.
347 
348 
349 ## Evidence File Format
350 Evidence is specified in a separate file. The evidence file consists of a single
351 line. The line will begin with the number of observed variables in the sample,
352 followed by pairs of variable and its observed value. The indexes correspond to
353 the ones implied by the original problem file.
354 
355 If, for our example Markov network, `Y` has been observed as having its first
356 value and `Z` with its second value, the evidence file would contain the
357 following line:
358 
359  2 1 0 2 1
360 
361 ## Query File Format
362 Query variables for Marginal MAP inference are specified in a separate file.
363 The query file consists of a single line. The line will begin with the number of
364 query variables, followed by the indexes of the query variables. The indexes
365 correspond to the ones implied by the original problem file.
366 
367 If, for our example Markov network, we want to use `Y` as the query variable
368 the query file would contain the following line:
369 
370  1 1
371 
372 ## Output File Format
373 The first line must contain only the task solved: `PR|MAP|MAR|MMAP`. The rest
374 of the file will contain the solution for the task. Solvers can write more then
375 one solution by writing `-BEGIN-` at the head of the new solution. In the
376 example below the task we choose is `PR`. We have two solutions. The format of
377 the `<SOLUTION>` part will be described below.
378 
379  PR
380  <SOLUTION>
381  -BEGIN-
382  <SOLUTION>
383 
384 The solution format are as follows depending on the task:
385 
386 ### Partition function `PR`
387 Line with the value of the log10 of the partition function. For example, an
388 approximation `log10 Pr(e) = -0.2008`which is known to be an upper bound may
389 have a solution line:
390 
391  -0.2008
392 ### Maximum aposteriori `MAP`
393 A space separated line that includes:
394 * the number `n` of model variables, and
395 * the MAP instantiation, a list of value indices for all `n` variables.
396 
397 For example, an input model with 3 binary variables may have a solution line:
398 
399  3 0 1 0
400 
401 ### Marginals `MAR`
402 A space separated line that includes:
403 * the number of variables `n` in the model, and
404 * a list of marginal approximations of all the variables. For each variable
405 its cardinality is first stated, then the probability of each state is stated.
406 The order of the variables is the same as in the model, all data is space
407 separated.
408 
409 For example, a model with `3` variables, with cardinalities of `2`, `2`, `3`
410 respectively. The solution might look like this:
411 
412  3 2 0.1 0.9 2 0.3 0.7 3 0.2 0.2 0.6
413 
414 ### Marginal MAP `MMAP`
415 A space separated line that includes:
416 * the number `q` of query (or MAP) variables, and
417 * their most probable instantiation, a list of variable value pairs for all `q`
418 variables.
419 
420 For example, if the solution is an assignment of `0`, `1` and `0` to three query
421 variables indexed by `2` `3` and `4` respectively, the solution will look as follows:
422 
423  3 2 0 3 1 4 0
